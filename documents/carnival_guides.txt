Hidden Tests
Hidden test cases will be used. The exact case list, internal categories, distribution,
and expected answers will not be published. Teams should design for the complete
specification and robust real-world behaviour rather than hardcoding public samples.
Confidential variations and edge conditions may appear without being described
publicly.

Evaluation Principle
The preliminary round selects teams that can build a safe, reliable, evidence-
grounded AI/API service under time pressure.

Flashy UI alone will not win. Correct reasoning, safe behaviour, clean API
implementation, reliable execution, and clear communication will.

Resource Policy
Poridhi resources are provided as support, not as a restriction. Teams may deploy
anywhere they want as long as the submitted API is reachable and judgeable.

API Submission Rule
The judge should be able to call the following endpoints from the submitted base
URL:

    GET https://your-service-url.com/health
    POST https://your-service-url.com/[main-endpoint]
REQUIREMENTS

    No login, dashboard access, manual approval, or private network access
    should be required for the judge.
    The service must accept JSON input and return JSON output.
    Use the exact endpoint names specified in the official Problem Statement.
    The service should remain reachable during the evaluation window.
